요청하면 코드 작성과 GitHub 저장까지 끝납니다. Vercel만 한 번 연결하면 제출용 링크가 생성됩니다.

처음 보는 용어가 많다면
아래 4가지만 알면 끝까지 무리 없이 따라오실 수 있습니다. 지금 외우실 필요는 없으니 한번 훑어보고 넘어가시면 됩니다.
웹 Claude Code claude.ai/code
설치 없이 브라우저에서 대화만으로 코딩하는 AI입니다. "이런 웹페이지 만들어줘"라고 한국어로 요청하면 코드를 대신 작성하고, 연결된 GitHub 저장소에 자동으로 저장까지 해 줍니다.
쉽게 말하면, 코드를 대신 작성하고 정리·저장까지 해 주는 만능 비서입니다. 말로 요청만 하면 됩니다.
index.html / 정적 웹페이지
브라우저에서 바로 열리는 웹페이지 파일 한 개입니다. 글, 그림, 버튼, 슬라이더 같은 인터랙션이 이 안에 모두 들어갑니다.
쉽게 말하면, 한 장짜리 완성된 전단지입니다. 이 파일만 있으면 웹사이트가 됩니다.
GitHub / Repository(저장소)
코드 파일을 인터넷에 보관하는 창고입니다. 저장소(repository)는 그 안의 프로젝트 폴더 하나입니다. Claude Code가 만든 코드는 여기에 자동으로 저장됩니다.
쉽게 말하면, 코드를 올려두는 구글 드라이브입니다. 저장소는 폴더 하나에 해당합니다.
Vercel / 배포(deploy)
GitHub에 올린 코드를 실제 인터넷 주소(URL)로 만들어 누구나 접속할 수 있게 해 주는 서비스입니다. 이 과정을 '배포'라고 합니다.
쉽게 말하면, 내 전단지를 거리에 붙여 모두가 볼 수 있게 하는 것입니다.
딱 3개만 준비합니다
준비물 (전부 무료)
시작 전에 계정 3개만 만들어 두시면 됩니다. 설치할 프로그램은 없고, 브라우저만 있으면 충분합니다.
  • Claude 계정 — claude.ai/code (Claude Code 사용. 본선팀 제공 플랜으로 로그인)
  • GitHub 계정 가입 — github.com (코드 저장, Claude Code와 연결)
  • Vercel 계정 가입 — vercel.com (GitHub 계정으로 바로 로그인 가능)
설치할 게 하나도 없습니다

터미널이나 Node.js 같은 프로그램은 설치하지 않아도 됩니다. 세 사이트 모두 브라우저에서 로그인만 하면 됩니다.

1단계
웹 Claude Code 시작하기
claude.ai/code에 접속해 GitHub 저장소를 연결하는 단계입니다. 한 번만 설정해 두면 이후부터는 대화만으로 코딩과 저장이 자동으로 진행됩니다.
1

GitHub 저장소 먼저 만들기

  1. github.com 로그인 후 우측 상단 + → New repository
  2. 이름 입력 (예: weather-project), Private 선택
  3. Create repository 클릭
저장소 이름은 나중에 Vercel 주소가 됩니다. 팀 프로젝트 이름으로 지어두세요.
2

claude.ai/code 접속 후 GitHub 연결

  1. claude.ai/code 접속 → 제공받은 계정으로 로그인
  2. 입력창 아래쪽의 "+ 레포 선택..." 버튼 클릭
  3. 목록에서 방금 만든 저장소(예: weather-project) 선택
  4. 선택이 끝나면 버튼 자리에 저장소 이름과 브랜치 이름(예: master)이 표시됩니다
이 연결을 한 번만 해두면 이후 Claude Code가 코드를 짤 때마다 GitHub에 자동으로 저장합니다.
2단계
Claude Code로 만들기
여기가 핵심입니다. 만들고 싶은 내용을 한국어로 설명하면 Claude가 코드를 작성하고 GitHub에 자동으로 저장합니다. 요청하실 때 ① 무엇을 보여줄지 ② 어떻게 조작할지 ③ 파일 이름(index.html), 이 세 가지만 포함하면 됩니다.
예시 대화
기온과 풍속 슬라이더로 체감온도를 보여주는 웹페이지를 index.html 하나로 만들어줘. 기상청 체감온도 공식을 써줘.
슬라이더를 움직이면 체감온도가 실시간으로 바뀌는 index.html을 만들었습니다. GitHub에 저장했어요.
배경색을 하늘색으로 바꾸고, 모바일에서도 잘 보이게 해줘.
배경색과 반응형 레이아웃을 적용했습니다. 변경 사항을 GitHub에 저장했어요.
P

바로 쓰는 프롬프트 모음

대괄호 부분만 우리 팀 주제로 바꾸세요.

  • 기본형 — "[기온]과 [풍속] 슬라이더로 [체감온도]를 보여주는 웹페이지를 index.html 하나로 만들어줘."
  • 응용형 — "[강수량] 슬라이더를 조절하면 [홍수 위험도]가 바뀌는 인터랙티브 웹페이지를 초등학생도 이해하게 만들어줘. index.html 하나로."
  • 디자인 — "방금 페이지를 더 보기 좋게 꾸며줘. 파란 하늘색 계열, 모바일에서도 잘 보이게."
  • 오류 해결 — "방금 페이지에서 [오류 메시지]가 나와. 원인을 알려주고 고쳐줘."
팁 한 번에 완벽하지 않아도 됩니다. "여기를 이렇게 바꿔줘"라고 계속 대화하며 다듬으세요. 수정할 때마다 GitHub에 자동 저장됩니다.
3단계
CLAUDE.md 설정하기
CLAUDE.md는 Claude Code에게 "이 프로젝트에서는 이렇게 해 달라"고 미리 일러두는 설명서 파일입니다. 코딩 규칙과 설정을 여기에 적어 두면 매번 설명하지 않아도 됩니다. 아래 내용을 복사해 저장소에 넣으시면 됩니다 (운영팀 제공).
CLAUDE.md가 뭔가요?

Claude Code가 프로젝트를 열 때 가장 먼저 읽는 파일입니다. 여기에 적어 둔 내용은 매 대화마다 자동으로 적용됩니다. "파일 이름은 index.html로 고정해 줘", "한국어로 답해 줘" 같은 규칙을 넣어 두면 됩니다.

1

CLAUDE.md 복사해서 저장소에 추가하기

  1. 아래 CLAUDE.md 내용을 전체 복사합니다.
  2. GitHub 저장소에서 Add file → Create new file 클릭
  3. 파일 이름을 CLAUDE.md로 입력하고 복사한 내용을 붙여넣기 → Commit changes
파일 이름은 반드시 CLAUDE.md (대문자)로, 위치는 저장소 최상단 폴더입니다.
CLAUDE.md (복사용)# CLAUDE.md ## Git 규칙 (필수) - main 브랜치에 직접 푸시하지 않습니다. 작은 수정도 예외 없음. - 모든 변경은 새 브랜치 → PR(Pull Request)로 머지합니다. - 팀원이 있으면: 팀원 1명 이상이 리뷰·승인한 뒤 머지 - 혼자(1인 팀)면: PR을 올려 변경 내용(diff)을 스스로 확인한 뒤 머지 - main에는 항상 정상 동작하는 코드만 둡니다.
4단계
Vercel로 배포하기
GitHub 저장소만 연결하면 인터넷 주소가 자동으로 만들어집니다.
1

GitHub로 로그인 후 프로젝트 가져오기

  1. vercel.com 접속 → Continue with GitHub
  2. Add New → Project 클릭
  3. 방금 만든 저장소(weather-windchill) 선택 → Import
2

Application Preset 설정

Import 직후 설정 화면이 열립니다. Framework Preset 항목에서 프로젝트에 맞는 항목을 선택합니다.

내 프로젝트 유형 선택할 Preset
HTML/CSS/JS 파일만 있는 경우 (index.html 중심) Other
Astro로 만든 경우 Astro
Next.js로 만든 경우 Next.js
Vite로 만든 경우 Vite
모르겠으면 Other — index.html을 직접 만들었다면 Other를 선택하세요. Preset이 틀리면 빌드가 실패합니다.
3

배포 후 링크 받기

  1. 설정 확인 후 Deploy 클릭
  2. 1~2분 뒤 weather-windchill.vercel.app 같은 주소 생성
  3. 이 주소가 제출용 링크입니다
편한 점 GitHub에서 index.html을 수정하면 Vercel이 자동으로 다시 배포합니다. 따로 재배포할 필요가 없습니다.
배포 후 확인

휴대폰이나 다른 컴퓨터에서도 링크가 열리는지, 슬라이더 같은 인터랙티브 기능이 정상 동작하는지 꼭 확인해 주세요.

5단계
제출하기
제출할 때는 아래 3가지를 모두 준비해야 합니다. Vercel 링크 하나만 제출하면 안 되니 주의하시기 바랍니다.
1

제출 항목 3가지

  • Vercel 배포 URL — 누구나 접속할 수 있는 웹 주소 (예: weather-project.vercel.app)
  • GitHub 저장소 URL — 코드가 저장된 저장소 주소 (예: github.com/팀명/weather-project). 커밋 히스토리가 남아 있어야 합니다.
  • Claude 대화 내역 — AI 활용 내역 확인용. 운영팀 안내에 따라 세션 링크 또는 스크린샷을 첨부합니다.
GitHub 커밋 히스토리는 표절·저작권 검증에 사용됩니다. 개발 과정 전체가 남도록 중간중간 저장하세요.
2

제출 전 체크리스트

  • Vercel URL이 다른 기기에서도 열리는가
  • GitHub 저장소가 Private으로 설정되어 있는가
  • 커밋 히스토리가 개발 과정 전체를 포함하는가
  • 사용한 외부 리소스(이미지, 폰트, 오픈소스)의 출처와 라이선스를 README에 명시했는가
팀이라면
팀 협업: GitHub PR 사용법
팀원이 각자 다른 기능을 개발할 때는 브랜치를 나눠 작업하고 PR(Pull Request)로 합칩니다. Claude Code에게 요청하면 브랜치 생성부터 PR 올리기까지 대신 처리해 줍니다. 각 단계마다 아래 프롬프트를 그대로 복사해 사용하시면 됩니다.
PR이 뭔가요?

Pull Request — 내가 만든 브랜치의 변경 내용을 팀 메인 코드에 합치자고 요청하는 것입니다. 팀원이 코드를 검토하고 승인하면 합쳐집니다. 충돌 없이 협업하는 표준 방식입니다.

1

브랜치 만들기

기능 개발을 시작할 때 main과 분리된 브랜치를 먼저 만듭니다. 팀원끼리 각자 다른 브랜치에서 작업하면 서로 방해받지 않습니다.

Claude Code 프롬프트 예시
새 브랜치를 만들어줘. 브랜치 이름은 feature/wind-chart 로 해줘.
feature/wind-chart 브랜치를 만들고 이동했습니다. 이제 이 브랜치에서 작업하면 됩니다.
지금 어느 브랜치에서 작업 중인지 알려줘.
현재 feature/wind-chart 브랜치에서 작업 중입니다.
브랜치 이름 규칙 feature/기능명 형태로 짓는 게 일반적입니다. 예: feature/typhoon-map, feature/rainfall-slider, fix/api-error
2

기능 개발하기

브랜치를 만든 뒤 이 브랜치 위에서 기능을 개발합니다. 평소처럼 Claude Code에게 말로 요청하면 됩니다.

Claude Code 프롬프트 예시
바람 방향을 0~360도 슬라이더로 조절하면 화살표가 회전하는 인터랙티브 컴포넌트를 만들어줘. index.html에 추가해줘.
바람 방향 슬라이더와 회전 화살표를 index.html에 추가했습니다.
화살표 색깔을 풍속에 따라 파랑(약)→주황(보통)→빨강(강)으로 바꿔줘.
풍속에 따라 색상이 바뀌도록 수정했습니다. 5m/s 이하 파랑, 10m/s 이하 주황, 그 이상 빨강으로 적용했어요.
3

커밋하기

작업한 내용을 중간중간 저장(커밋)합니다. 커밋은 "이 시점까지 작업한 스냅샷"을 남기는 것입니다. 나중에 문제가 생겼을 때 이 시점으로 되돌아올 수 있습니다.

Claude Code 프롬프트 예시
지금까지 변경한 내용을 커밋해줘. 메시지는 "바람 방향 화살표 컴포넌트 추가"로 해줘.
변경 사항을 커밋했습니다. 메시지: "바람 방향 화살표 컴포넌트 추가"
지금까지 커밋 기록을 보여줘.
총 3개의 커밋이 있습니다: (1) 초기 세팅 (2) 체감온도 슬라이더 추가 (3) 바람 방향 화살표 컴포넌트 추가
팁 큰 작업이 끝날 때마다 커밋하세요. 커밋 메시지는 "무엇을 했는지" 한 줄로 씁니다. 커밋 히스토리는 심사 및 표절 검증에 활용됩니다.
4

PR 올리기

기능 개발이 끝나면 팀원에게 검토를 요청하는 PR을 올립니다. Claude Code가 제목과 설명까지 자동으로 작성해줍니다.

Claude Code 프롬프트 예시
feature/wind-chart 브랜치를 main에 합치는 PR을 올려줘. 제목은 "바람 방향 시각화 기능 추가"로 해줘.
PR을 생성했습니다. 제목: "바람 방향 시각화 기능 추가". GitHub에서 확인하실 수 있습니다.
PR 설명에 "기온/풍속 슬라이더에 더해 바람 방향 화살표를 추가했고, 풍속에 따라 색이 바뀝니다"라고 써줘.
PR 설명을 업데이트했습니다.
5

리뷰 요청하기

PR을 올린 뒤에는 팀원에게 리뷰를 요청해야 합니다. Claude Code에게 팀원 GitHub 아이디를 알려주면 리뷰어로 지정해줍니다.

Claude Code 프롬프트 예시
이 PR에 영욱님 리뷰를 요청해줘.
PR에 ywkim님을 리뷰어로 추가했습니다.

또는 GitHub 웹에서 직접 요청하기

  1. GitHub에서 PR 페이지를 엽니다
  2. 오른쪽 사이드바 Reviewers 옆의 톱니바퀴(⚙️) 아이콘 클릭
  3. 드롭다운에서 리뷰를 요청할 팀원 선택
  4. 선택하는 즉시 그 팀원에게 리뷰 요청 알림이 전송됩니다

Push할 때 SSH 키 에러가 날 때

새 브랜치를 GitHub에 올리려고 할 때 "Permission denied (publickey)" 같은 에러가 날 수 있습니다. 웹 Claude Code는 개인 SSH 키를 전혀 쓰지 않고 GitHub App 연결(설치)로 인증하기 때문에, 이 에러는 거의 항상 App 설치·권한 문제입니다.

  1. GitHub App이 이 저장소에 설치되어 있지 않은 경우 — Step 1에서 다른 저장소만 연결했거나, 저장소를 새로 만든 뒤 App 연결을 갱신하지 않은 경우입니다.
  2. App 권한이 읽기 전용인 경우 — 연결 당시 쓰기(write) 권한을 허용하지 않으면 push 단계에서 거부됩니다.
A

GitHub App 설치·권한 다시 연결하기

  1. GitHub 우측 상단 프로필 → Settings → 좌측 메뉴 Applications → Installed GitHub Apps
  2. Claude(또는 연결된 GitHub App) 옆 Configure 클릭
  3. Repository access에서 작업 중인 저장소가 포함되어 있는지 확인 (없으면 Select repositories로 추가)
  4. 권한 목록에서 Contents: Read and write, Pull requests: Read and write가 허용되어 있는지 확인
  5. 변경 후 claude.ai/code로 돌아와 "+ 레포 선택..." 버튼으로 저장소를 다시 선택
권한을 바꾼 직후에는 바로 반영되지 않을 수 있습니다. claude.ai/code 페이지를 새로고침한 뒤 저장소를 다시 선택해보세요.
Claude Code 프롬프트 예시
브랜치를 push하려는데 "Permission denied (publickey)" 에러가 나. 원인이 뭔지 확인해줘.
현재 origin 주소를 확인했습니다. GitHub App 연결에 이 저장소가 포함되어 있지 않은 것으로 보입니다. GitHub Settings → Applications에서 저장소 접근 권한을 추가한 뒤, "+ 레포 선택..."으로 다시 연결해주세요.
핵심 웹 환경에서는 SSH 키를 직접 만들거나 암호를 바꿀 필요가 없습니다. 에러 메시지에 "SSH"나 "publickey"가 보여도, 해결은 항상 GitHub App 설치 화면에서 저장소·권한을 다시 확인하는 것입니다.
팀 작업 권장 흐름

기능별로 브랜치 나누기 → 각자 개발 후 PR → 팀원 1명 이상 리뷰 → 머지 → Vercel 자동 배포. main 브랜치에는 항상 정상 동작하는 코드만 두는 것이 원칙입니다.
