웹 Claude Code로 만든다

이 대회의 결과물은 누구나 웹에서 접속해 직접 조작하는 인터랙티브 도구입니다. 웹브라우저에서 Claude Code(claude.ai/code)에게 한국어로 요청하면 코드를 만들고 GitHub에 자동 저장까지 해줍니다. 그 뒤 Vercel로 한 번 연결하면 제출용 링크가 완성됩니다.

How It Works
무엇을 만드나요?
기상·기후 데이터를 학습자가 직접 조작하며 배우는 웹 도구를 만듭니다. 태풍 시뮬레이터, 예보관 체험, 기압·바람 인터랙티브 맵처럼요. 서버나 앱 없이 웹페이지 파일 하나(index.html)로도 충분히 제출할 수 있습니다.

한 줄 요약

Claude Code에게 말로 시키면 코드 작성과 GitHub 저장까지 끝. Vercel만 한 번 연결하면 제출용 링크가 나옵니다.

Before You Start
처음 보는 용어가 많다면
아래 5개만 알면 끝까지 따라올 수 있습니다. 지금 다 외울 필요는 없어요.
웹 Claude Code claude.ai/code
설치 없이 웹브라우저에서 대화만으로 코딩하는 AI 도구. 한국어로 "이런 웹페이지 만들어줘"라고 요청하면 코드를 대신 작성하고, 연결된 GitHub 저장소에 자동으로 저장까지 해줍니다.
비유: 코드를 대신 써주고 정리·저장까지 해주는 만능 비서. 말로 설명만 하면 됩니다.
index.html / 정적 웹페이지
브라우저에서 바로 열리는 웹페이지 파일 한 개. 글, 그림, 버튼, 슬라이더 같은 인터랙션이 이 안에 다 들어갑니다.
비유: 한 장짜리 완성된 전단지. 이 파일만 있으면 웹사이트가 됩니다.
GitHub / Repository(저장소)
코드 파일을 인터넷에 보관하는 창고. 저장소(repository)는 그 안의 프로젝트 폴더 하나입니다. 웹 Claude Code가 만든 코드는 여기에 자동으로 저장됩니다.
비유: 코드를 올려두는 구글 드라이브. 저장소 = 폴더 하나.
Vercel / 배포(deploy)
GitHub에 올린 코드를 실제 인터넷 주소(URL)로 만들어 누구나 접속하게 해주는 서비스. 이 과정을 배포라고 합니다.
비유: 내 전단지를 거리에 붙여서 모두가 볼 수 있게 하는 것.
공공데이터 API
기상청 공공데이터포털(data.go.kr)에서 실제 기온·강수 같은 데이터를 코드로 불러오는 통로. 필수는 아니며, 공식이나 예시 데이터로도 충분합니다.
비유: 기상청 창고에서 데이터를 꺼내오는 출입증.
Checklist
준비물 (모두 무료)
시작 전에 계정 3개만 만들어 두세요. 설치할 프로그램은 없고, 웹브라우저만 있으면 됩니다.
  • Claude 계정 — claude.ai/code (Claude Code 사용. 본선팀 제공 플랜으로 로그인)
  • GitHub 계정 가입 — github.com (코드 저장, Claude Code와 연결)
  • Vercel 계정 가입 — vercel.com (GitHub 계정으로 바로 로그인 가능)
설치할 게 없습니다

터미널, Node.js 같은 프로그램 설치는 필요 없습니다. 세 사이트 모두 웹브라우저에서 로그인만 하면 됩니다.

Step 1
웹 Claude Code 시작하기
claude.ai/code에 접속해 GitHub 저장소를 연결하는 단계입니다. 한 번만 설정해두면 이후부터는 대화만으로 코딩과 저장이 자동으로 됩니다.
1

GitHub 저장소 먼저 만들기

  1. github.com 로그인 후 우측 상단 + → New repository
  2. 이름 입력 (예: weather-project), Public 선택
  3. Create repository 클릭
저장소 이름은 나중에 Vercel 주소가 됩니다. 팀 프로젝트 이름으로 지어두세요.
2

claude.ai/code 접속 후 GitHub 연결

  1. claude.ai/code 접속 → 제공받은 계정으로 로그인
  2. 입력창 아래쪽의 "+ 레포 선택..." 버튼 클릭
  3. 목록에서 방금 만든 저장소(예: weather-project) 선택
  4. 선택이 끝나면 버튼 자리에 저장소 이름과 브랜치 이름(예: master)이 표시됩니다
이 연결을 한 번만 해두면 이후 Claude Code가 코드를 짤 때마다 GitHub에 자동으로 저장합니다.
Step 2
Claude Code로 만들기
핵심 단계입니다. 만들고 싶은 걸 한국어로 설명하면 Claude가 코드를 짜고 GitHub에 자동 저장합니다. 요청할 때 (1) 무엇을 보여줄지 (2) 어떻게 조작할지 (3) 파일 이름(index.html), 이 3가지만 넣으면 됩니다.
예시 대화
기온과 풍속 슬라이더로 체감온도를 보여주는 웹페이지를 index.html 하나로 만들어줘. 기상청 체감온도 공식을 써줘.
슬라이더를 움직이면 체감온도가 실시간으로 바뀌는 index.html을 만들었습니다. GitHub에 저장했어요.
배경색을 하늘색으로 바꾸고, 모바일에서도 잘 보이게 해줘.
배경색과 반응형 레이아웃을 적용했습니다. 변경 사항을 GitHub에 저장했어요.
P

바로 쓰는 프롬프트 모음

대괄호 부분만 우리 팀 주제로 바꾸세요.

  • 기본형 — "[기온]과 [풍속] 슬라이더로 [체감온도]를 보여주는 웹페이지를 index.html 하나로 만들어줘."
  • 응용형 — "[강수량] 슬라이더를 조절하면 [홍수 위험도]가 바뀌는 인터랙티브 웹페이지를 초등학생도 이해하게 만들어줘. index.html 하나로."
  • 디자인 — "방금 페이지를 더 보기 좋게 꾸며줘. 파란 하늘색 계열, 모바일에서도 잘 보이게."
  • 데이터 연동 — "기상청 공공데이터포털의 [기온] API를 불러와 화면에 표시해줘. API 키는 내가 넣을 수 있게 표시해줘."
  • 오류 해결 — "방금 페이지에서 [오류 메시지]가 나와. 원인을 알려주고 고쳐줘."
팁 한 번에 완벽하지 않아도 됩니다. "여기를 이렇게 바꿔줘"라고 계속 대화하며 다듬으세요. 수정할 때마다 GitHub에 자동 저장됩니다.
Step 3
CLAUDE.md 설정하기
CLAUDE.md는 Claude Code에게 "이 프로젝트에서 지켜야 할 규칙"을 미리 알려주는 설명서 파일입니다. 기상청 API 키, 데이터 형식, 코딩 규칙을 여기 써두면 매번 설명하지 않아도 됩니다. 운영팀이 기상청 API용 CLAUDE.md 파일을 제공할 예정입니다.
CLAUDE.md가 뭔가요?

Claude Code가 프로젝트를 열 때 가장 먼저 읽는 파일입니다. 여기 써둔 내용은 매 대화마다 자동으로 적용됩니다. "기상청 API는 이렇게 써", "파일 이름은 index.html로 고정해" 같은 규칙을 넣어두면 됩니다.

1

운영팀 제공 CLAUDE.md 저장소에 추가하기

  1. 운영팀이 제공한 CLAUDE.md 파일 다운로드
  2. GitHub 저장소에서 Add file → Upload files 클릭
  3. CLAUDE.md 파일을 드래그 앤 드롭 → Commit changes
파일 이름은 반드시 CLAUDE.md (대문자)로 두세요. 위치는 저장소 최상단 폴더입니다.
2

기상청 API 키 넣기

  1. 공공데이터포털(data.go.kr)에서 발급받은 API 키를 준비
  2. CLAUDE.md 파일 안의 API_KEY 항목에 붙여넣기
  3. Claude Code에서 "기상청 API로 현재 기온 불러와줘"라고 요청하면 바로 연동됩니다
주의 API 키는 GitHub에 공개된 파일에 직접 노출하지 마세요. Claude Code 내에서만 입력하거나, 운영팀 안내에 따라 처리하세요.
Step 4
Vercel로 배포하기
GitHub 저장소를 연결하면 인터넷 주소가 자동으로 생성됩니다.
1

GitHub로 로그인 후 프로젝트 가져오기

  1. vercel.com 접속 → Continue with GitHub
  2. Add New → Project 클릭
  3. 방금 만든 저장소(weather-windchill) 선택 → Import
2

배포 후 링크 받기

  1. 별도 설정 없이 Deploy 클릭 (정적 웹페이지는 자동 인식)
  2. 1~2분 뒤 weather-windchill.vercel.app 같은 주소 생성
  3. 이 주소가 제출용 링크입니다
편한 점 GitHub에서 index.html을 수정하면 Vercel이 자동으로 다시 배포합니다. 따로 재배포할 필요가 없어요.
배포 후 확인

휴대폰이나 다른 컴퓨터에서도 링크가 열리는지, 슬라이더 등 인터랙티브 기능이 잘 작동하는지 꼭 확인하세요.

Step 5
제출하기
완성된 결과물을 제출할 때는 아래 3가지를 모두 준비합니다. Vercel 링크 하나만 있으면 안 되니 주의하세요.
1

제출 항목 3가지

  • Vercel 배포 URL — 누구나 접속할 수 있는 웹 주소 (예: weather-project.vercel.app)
  • GitHub 저장소 URL — 코드가 저장된 저장소 주소 (예: github.com/팀명/weather-project). 커밋 히스토리가 남아 있어야 합니다.
  • Claude 대화 내역 — AI 활용 내역 확인용. 운영팀 안내에 따라 세션 링크 또는 스크린샷을 첨부합니다.
GitHub 커밋 히스토리는 표절·저작권 검증에 사용됩니다. 개발 과정 전체가 남도록 중간중간 저장하세요.
2

제출 전 체크리스트

  • Vercel URL이 다른 기기에서도 열리는가
  • GitHub 저장소가 Public으로 설정되어 있는가
  • 커밋 히스토리가 개발 과정 전체를 포함하는가
  • 사용한 외부 리소스(이미지, 폰트, 오픈소스)의 출처와 라이선스를 README에 명시했는가
  • 기상청 API 키가 코드에 노출되어 있지 않은가

Team Collaboration
팀 협업: GitHub PR 사용법
팀원이 각자 다른 기능을 개발할 때는 브랜치를 나눠서 작업하고 PR(Pull Request)로 합칩니다. Claude Code에게 말로 시키면 브랜치 생성부터 PR 올리기까지 전부 대신 해줍니다. 각 단계마다 아래 프롬프트를 그대로 복사해서 쓰세요.
PR이 뭔가요?

Pull Request — 내가 만든 브랜치의 변경 내용을 팀의 메인 코드에 합치자고 요청하는 것입니다. 팀원이 코드를 검토하고 OK하면 합쳐집니다. 충돌 없이 협업하는 표준 방식입니다.

1

브랜치 만들기

기능 개발을 시작할 때 main과 분리된 브랜치를 먼저 만듭니다. 팀원끼리 각자 다른 브랜치에서 작업하면 서로 방해받지 않습니다.

Claude Code 프롬프트 예시
새 브랜치를 만들어줘. 브랜치 이름은 feature/wind-chart 로 해줘.
feature/wind-chart 브랜치를 만들고 이동했습니다. 이제 이 브랜치에서 작업하면 됩니다.
지금 어느 브랜치에서 작업 중인지 알려줘.
현재 feature/wind-chart 브랜치에서 작업 중입니다.
브랜치 이름 규칙 feature/기능명 형태로 짓는 게 일반적입니다. 예: feature/typhoon-map, feature/rainfall-slider, fix/api-error
2

기능 개발하기

브랜치를 만든 뒤 이 브랜치 위에서 기능을 개발합니다. 평소처럼 Claude Code에게 말로 요청하면 됩니다.

Claude Code 프롬프트 예시
바람 방향을 0~360도 슬라이더로 조절하면 화살표가 회전하는 인터랙티브 컴포넌트를 만들어줘. index.html에 추가해줘.
바람 방향 슬라이더와 회전 화살표를 index.html에 추가했습니다.
화살표 색깔을 풍속에 따라 파랑(약)→주황(보통)→빨강(강)으로 바꿔줘.
풍속에 따라 색상이 바뀌도록 수정했습니다. 5m/s 이하 파랑, 10m/s 이하 주황, 그 이상 빨강으로 적용했어요.
3

커밋하기

작업한 내용을 중간중간 저장(커밋)합니다. 커밋은 "이 시점까지 작업한 스냅샷"을 남기는 것입니다. 나중에 문제가 생겼을 때 이 시점으로 되돌아올 수 있습니다.

Claude Code 프롬프트 예시
지금까지 변경한 내용을 커밋해줘. 메시지는 "바람 방향 화살표 컴포넌트 추가"로 해줘.
변경 사항을 커밋했습니다. 메시지: "바람 방향 화살표 컴포넌트 추가"
지금까지 커밋 기록을 보여줘.
총 3개의 커밋이 있습니다: (1) 초기 세팅 (2) 체감온도 슬라이더 추가 (3) 바람 방향 화살표 컴포넌트 추가
팁 큰 작업이 끝날 때마다 커밋하세요. 커밋 메시지는 "무엇을 했는지" 한 줄로 씁니다. 커밋 히스토리는 심사 및 표절 검증에 활용됩니다.

충돌(Conflict)이 생겼을 때

두 팀원이 같은 파일의 같은 부분을 동시에 수정하면 충돌이 생깁니다. 겁먹지 말고 Claude Code에게 그대로 알려주면 됩니다.

Claude Code 프롬프트 예시

두 내용 다 살려서 합쳐줘.
두 변경 사항을 합쳐서 충돌을 해결했습니다. 커밋했어요.

API 키
기상청 API 키 발급 가이드
기상청 실데이터를 쓰려면 API 키가 필요합니다. 무료지만 발급 과정에서 막히는 경우가 많습니다. 실제 겪은 어려움을 기준으로 정리했습니다.
주의: 승인 후 실제 API 게이트웨이 반영까지 1~2시간 걸릴 수 있습니다. "승인" 표시가 떠도 바로 호출이 안 될 수 있으니 여유 있게 미리 신청하세요.
1

data.go.kr — 동네예보 API (단기·중기)

  1. data.go.kr 로그인 후 "기상청 단기예보" 검색
  2. 오픈API 탭 선택 (파일데이터 아님)
  3. 기상청_단기예보 조회서비스 → 활용신청
  4. 활용목적: "기상 교육 콘텐츠 개발" 입력, 상세기능 단기예보조회 체크
  5. 승인 후 일반 인증키(Encoding) 복사 → Vercel 환경변수 KMA_API_KEY에 저장
엔드포인트https://apis.data.go.kr/1360000/VilageFcstInfoService_2.0/getVilageFcst
2

apihub.kma.go.kr — 기상청 자체 허브 (더 많은 API)

  1. apihub.kma.go.kr 가입 및 로그인
  2. 원하는 API 페이지에서 API 활용신청 버튼 클릭
  3. 발급된 authKey 복사 → 환경변수에 저장
  4. data.go.kr보다 API 종류가 많음 (레이더, 특보, 태풍 등)
data.go.kr과 apihub.kma.go.kr은 별개 시스템입니다. 키도 다르고 파라미터 이름도 다릅니다. (serviceKey vs authKey)
3

API 키를 코드에 직접 넣지 마세요

index.html에 API 키를 하드코딩하면 GitHub에 키가 노출됩니다. 반드시 Vercel 환경변수로 관리하세요.

Vercel 환경변수 설정vercel.com → 프로젝트 → Settings → Environment Variables → KMA_API_KEY = 발급받은 키 붙여넣기

FAQ
자주 묻는 질문
코딩을 전혀 못하는데 가능한가요?

가능합니다. 웹 Claude Code에 한국어로 요청하면 코드를 대신 짜줍니다. 이 가이드의 순서대로만 따라 하면 됩니다.

백엔드 서버가 꼭 필요한가요?

아닙니다. 브라우저에서 동작하는 웹페이지(index.html) 하나로 충분합니다. 서버 없이도 기상청 API를 불러올 수 있습니다.

데이터는 어디서 가져오나요?

공식을 직접 넣어도 되고(예: 체감온도 공식), 기상청 공공데이터포털(data.go.kr)의 API를 활용해도 됩니다. 운영팀이 제공하는 CLAUDE.md에 API 사용 방법이 안내되어 있습니다.

팀원과 함께 작업하려면?

GitHub 저장소 Settings → Collaborators에서 팀원을 추가하면 됩니다. 각자 Claude Code에서 같은 저장소에 연결해 작업할 수 있습니다.

기술·도구 문의는 어디에 하나요?

Slack의 #질문-공통 채널에 올려주세요. 도구·기술 문의는 운영팀이 직접 응대하고, 기상청 API 관련 문의는 운영팀 접수 후 기상청 담당자를 통해 회신합니다. 정규 응대일은 매주 공지를 확인해주세요.

Claude Code 계정은 어떻게 받나요?

본선 진출 15팀에게 운영팀이 이메일로 계정을 발급합니다. 계정 접속 문제는 Slack #질문-공통 채널 또는 긴급 시 운영팀에 직접 연락하세요.

AI가 만든 코드를 그대로 제출해도 되나요?

됩니다. 이 대회는 AI 활용을 권장합니다. 단, 다른 팀이나 외부 출처의 코드를 그대로 복사하는 것은 표절로 처리됩니다. AI로 생성했더라도 우리 팀의 아이디어와 구성이 담겨 있어야 합니다.
