Slack · 초대장 한 번으로 시작

멘토에게 질문하고 공지를 받는 공간이에요. 운영팀이 보내는 초대 메일을 수락하면 바로 참여할 수 있고, 채널은 목적별로 나뉘어 있으니 질문 성격에 맞게 골라 쓰면 됩니다.

Join

초대받고 가입하기

먼저 확인하세요 — 참여는 이메일 초대로만

참가자는 이메일 초대로만 워크스페이스에 참여합니다. 운영팀이 참가자 이메일을 모은 뒤, 그 주소로 초대 메일을 보내드려요. 직접 가입 신청하거나 링크를 찾을 필요는 없습니다.

가입 순서

  1. 운영팀이 보낸 Slack 초대 메일을 엽니다.
  2. 메일 안의 참여(Join 또는 초대수락) 버튼을 누릅니다.
  3. 이름·비밀번호를 설정하면(또는 Google 계정 등으로 계속) 가입이 끝납니다.
  4. 워크스페이스 26k-weatherhackathon.slack.com에 입장하면 멘토·운영 채널이 보입니다.
데스크톱 앱 설치를 권장하지만 브라우저 로그인만으로도 쓸 수 있어요. 알림을 놓치지 않으려면 모바일 앱(iOS·Android)을 함께 설치하는 걸 권합니다.

질문하기

어느 채널에 쓸까요

질문 성격에 맞는 채널을 고르면 됩니다. 판단이 서지 않으면 운영_기타문의에 남겨 주세요. 운영팀이 안내해 드립니다.

  • 교육 콘텐츠·데이터 API 활용 → 멘토_교육_api관련
  • 대기·기상·기후 등 과학 개념 → 멘토_대기_과학
  • 기획·UX·콘텐츠 구성 → 멘토_서비스_기획
  • AI 도구·개발·구현 방법 → 멘토_ai기술_개발
  • 일정·마감 등 공식 공지 확인 → 운영_전체공지(읽기 중심)
  • 계정·환경·기타 운영 문의 → 운영_기타문의

이렇게 질문하면 좋아요

채널: 운영_기타문의
Claude 계정 초대 메일이 오지 않아요.
먼저 스팸함과 프로모션함을 확인해 주세요. 그래도 없으면 운영_기타문의에 가입 신청하신 이메일 주소를 남겨 주시면, 운영팀이 발송 내역을 확인해 다시 보내드립니다.
계정·환경·접속처럼 운영팀이 확인해 줄 수 있는 문제는 이렇게 운영 채널에서 빠르게 풀립니다.

이런 질문은 멘토가 대신 풀어주지 않아요

성격상 멘토_ai기술_개발처럼 보이지만, 팀이 직접 해결해야 하는 문제예요.

채널: 멘토_ai기술_개발
코드 오류를 어떻게 해결해야 하나요?
특정 코드의 오류를 대신 고쳐 드리지는 않아요. 개발 과정에서 팀이 스스로 해결해야 하는 부분이고, 문제 해결 능력도 평가 대상이기 때문입니다. 다만 "이 접근 방식이 맞는 방향인지", "어떤 도구·개념을 참고하면 좋을지"처럼 방향을 묻는 질문에는 멘토가 조언해 드릴 수 있어요.
멘토 채널은 팀의 문제를 대신 풀어주는 곳이 아니라, 방향과 개념을 함께 짚어주는 곳이에요. 구체적인 구현과 디버깅은 팀의 몫으로 남겨 두세요.
언제 답변받나요

평일 기준으로 질문에 최대 24시간 이내 답변하는 것을 목표로 합니다.

Tips

Slack 기본 사용 팁

  • 스레드로 답하기 — 한 질문에 답이 여러 개 이어질 때는 원글 하단의 회신(스레드)을 쓰면 채널이 깔끔해요.
  • 멘션 — @이름으로 특정 멘토·운영자를 부를 수 있어요. 급하지 않은 질문은 채널에 남겨두면 담당 멘토가 확인합니다.
  • 파일 첨부 — 스크린샷·로그를 바로 올릴 수 있어요. 단, 개인 API 키나 비밀값은 절대 올리지 마세요.
  • 검색 — 상단 검색창에서 키워드로 이전 답변을 찾아 중복 질문을 줄일 수 있어요.
