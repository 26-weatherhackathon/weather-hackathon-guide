01 TRACK 01 / BEFORE YOU START

처음 보는 용어가 많다면

가이드에 자주 나오는 핵심 용어 5개를 먼저 정리했습니다. 지금 모두 외울 필요는 없으며, 막히면 다시 돌아와 확인하면 됩니다.

Glossary

용어 사전

웹 Claude Code claude.ai/code
설치 없이 웹브라우저에서 대화만으로 코딩하는 AI 도구입니다. 한국어로 "이런 웹페이지를 만들어 달라"고 요청하면 코드를 대신 작성합니다. 처음에 GitHub 저장소를 한 번 연결해 두면(트랙 02 Step 1), 이후 작성한 코드가 그 저장소에 자동으로 저장됩니다.
비유하자면, 요청한 내용을 코드로 작성하고 저장까지 대신 처리해 주는 도구입니다.
index.html / 정적 웹페이지
브라우저에서 바로 열리는 웹페이지 파일 한 개입니다. 글, 그림, 버튼, 슬라이더 같은 인터랙션이 모두 이 안에 들어갑니다.
비유하자면, 한 장으로 완성된 전단지입니다. 이 파일만 있으면 웹사이트가 됩니다.
GitHub / Repository(저장소)
코드 파일을 인터넷에 보관하는 창고입니다. 저장소(repository)는 그 안의 프로젝트 폴더 하나에 해당합니다. 트랙 02 Step 1에서 저장소를 만들어 웹 Claude Code에 연결하면, 이후 작성한 코드가 이 저장소에 저장됩니다. 이 저장소는 개발용 작업 공간이며, 최종 소스 제출은 저장소 링크가 아니라 별도 구글 폼에 업로드합니다(트랙 02 Step 5).
비유하자면, 코드를 올려 두는 온라인 저장 공간이며, 저장소는 그 안의 폴더 하나입니다.
Vercel / 배포(deploy)
GitHub에 올린 코드를 실제 인터넷 주소(URL)로 만들어 누구나 접속할 수 있게 하는 서비스입니다. 이 과정을 배포라고 합니다.
비유하자면, 완성한 전단지를 거리에 붙여 누구나 볼 수 있게 하는 것과 같습니다.
PR / Pull Request
내 브랜치의 변경 내용을 팀의 메인 코드에 합치자고 요청하는 것입니다. 팀원이 검토(리뷰)한 뒤 승인하면 합쳐집니다. 자세한 내용은 트랙 03을 참고하세요.
비유하자면, "이 부분을 이렇게 바꿨는데 반영해도 될지" 확인을 요청하는 검토 요청서입니다.

Checklist

준비물 (모두 무료)

계정 3개를 미리 만들어 두세요. 웹 인터페이스에서는 설치할 프로그램이 없으며, 웹브라우저만 있으면 됩니다.

  • Claude 계정
    Claude Code 사용. claude.ai/code — 본선 진출 팀에 운영팀이 발급하는 팀 계정으로 로그인합니다.
  • GitHub 계정
    코드 저장소. github.com에서 가입한 뒤 Claude Code와 연결합니다.
  • Vercel 계정
    배포 서비스. vercel.com — GitHub 계정으로 바로 로그인할 수 있습니다.
웹 인터페이스는 설치가 없습니다

초보자용 웹 인터페이스에서는 터미널이나 Node.js 설치가 필요하지 않으며, 세 사이트 모두 브라우저 로그인만으로 시작합니다. 터미널·IDE 환경에 익숙하다면 그 환경으로 작업해도 됩니다.
