Security

API 키 보안 (필수)

API 키가 노출되면 실격입니다

API 키·토큰·비밀번호 같은 비밀값은 어떤 경우에도 코드에 그대로 노출되지 않도록 설계해야 합니다. GitHub에 올라간 코드와 배포된 웹페이지는 누구나 열어볼 수 있어, 노출된 키는 도용·과금 사고로 이어집니다. 키 노출이 확인되면 실격 처리됩니다.

하지 말 것 ✗이렇게 설계하세요 ✓
키를 HTML·JS 코드에 직접 써넣기키가 필요 없는 공개 데이터·API를 우선 사용
키를 GitHub에 커밋키가 꼭 필요하면 코드가 아닌 환경변수에 두고, 서버(서버리스 함수)를 거쳐 호출
키를 Slack·PR·스크린샷에 노출키에 사용량·도메인 제한을 걸어 피해 범위를 최소화
노출된 키를 그대로 두기실수로 올렸다면 즉시 키를 폐기(revoke)하고 새로 발급
막혔을 때

"이 API 키가 코드에 노출되지 않게 안전하게 쓰는 방법 알려줘"라고 Claude Code에게 물어보면 환경변수 설정과 호출 방식까지 안내해 줍니다.
