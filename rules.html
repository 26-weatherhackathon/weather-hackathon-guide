규정 · 저작권 & 보안

제출물은 저작권·라이선스 기준을 지켜야 합니다. 문제가 확인되면 규정에 따라 단계적으로 제재(소명 절차 포함)될 수 있으니 기준을 확인하세요.

저작권 가이드라인

표절 및 저작권 검증을 실시합니다

제출물은 팀 간·외부 출처 대비 표절 검증과 제공 계정의 개발 세션 확인을 거칩니다. AI 사용 자체는 문제되지 않으나, 다른 팀이나 외부 출처의 코드를 그대로 사용하는 것은 허용되지 않습니다.

AI로 코드를 생성했더라도 저작권·특허·보안에 대한 책임이 면제되지 않으며, 제출물에 대한 최종 책임은 참가자에게 있습니다.

기상청 자료를 이용할 때

기상청 홈페이지 자료는 공공누리(KOGL) 유형을 확인하고 출처를 구체적으로 표시해 이용합니다. 제1유형(출처표시)은 출처만 밝히면 자유이용이 가능하며, 제2·3·4유형은 상업적 이용·변형 등 제한사항을 확인해야 합니다. 자세한 내용은 기상청 저작권 보호 및 정책을 확인하세요.

Security

API 키·비밀값 보안

API 키가 노출되면 실격입니다

API 키를 비롯한 모든 비밀값(키·토큰·인증 정보 등)은 코드에 노출되지 않도록 다뤄야 합니다. 기상청 API 키뿐 아니라 외부 서비스에서 발급받은 어떤 키라도 마찬가지입니다. GitHub에 올라간 코드와 배포된 웹페이지는 누구나 열어볼 수 있어, 노출된 키는 도용·과금 사고로 이어질 수 있습니다. API 키가 노출되면 실격 처리되므로 보안에 유의하여 설계하세요.

다음과 같은 노출에 유의하세요.

  • ✕API 키·토큰을 HTML·JS 코드에 직접 써넣기
  • ✕키를 GitHub 저장소에 커밋
  • ✕키를 Slack·PR·스크린샷 등에 노출
  • ✕노출된 키를 그대로 방치
