# 데이터 활용 극대화 & 화려한 인터랙션 계획 (Data & Interaction Spec v1.0)

> 기상·기후 AI 해커톤 2026 출품작 · **"공공데이터 뽕 뽑기" + 인터랙티브 연출 설계**

> 그걸 **만지고 싶은 화려한 인터랙션**으로 만든다(체험형 25점·독창성 15점).

## 1. 가용 데이터 전수 인벤토리 (무엇이 있는가)

기상청은 두 창구로 데이터를 공개한다. **둘 다 쓴다.**

### 1.1 기상청 API허브 (apihub.kma.go.kr · authKey) — 12개 분야
원시·고해상도 데이터와 **영상 파일**까지 주는 본진. 우리 케이스 영상의 출처.

| 분야 | 우리가 쓰는 데이터 | 케이스 활용 | 단계 |
|---|---|---|---|
| **위성** | GK2A 16채널 — 적외(IR)·가시(VIS)·**수증기(WV)** + 산출물 46종 | 3채널 토글, 수증기로 호우 수분 유입 읽기 | S2 |
| **레이더** | HSR 합성영상 + 강수량(HSR) + **에코 이동속도** | 타임라인 스크럽 + 이동벡터 오버레이 | S2 |
| **지상관측** | ASOS(종관)·AWS(방재) 기온·강수·바람·습도·기압 | 멀티라인 추세 + 촘촘한 격자 핀 | S2 |
| **고층관측** | 레윈존데 연직 기온·이슬점·풍향풍속 | **SKEW-T 단열선도 인터랙티브** + CAPE/CIN/K-Index | S3 |
| **수치예보(일기도)** | GDAPS/LDAPS 지상·850·500hPa 일기도 이미지 시계열 | 종관 타임머신 — 전선 이동·기압골 접근 | S1 |
| **수치예보(격자)** | 단기예보 getVilageFcst 2개 발표시각 | KIM/UM 모델 불일치 비교 | S4 |
| **수치예보(앙상블)** | 앙상블 멤버 강수 예측 분포 | 스프레드 시각화 — 불확실성 체감 | S4 |
| **해양관측** | 부이 파고·수온·파주기 | (태풍 확장 케이스) 해상 상태 | — |
| **태풍** | 태풍 정보·예상경로 | (확장 케이스) 경로 콘 시각화 | — |
| 예특보/응용/항공/세계/지진화산 | (직접 사용 낮음) | 확장 여지 | — |

> 실제 엔드포인트 예시(확인됨):
> - 위성: `apihub.kma.go.kr/api/typ05/api/GK2A/LE1B/{채널}/FD/data?date=YYYYMMDDHHMM&authKey=…`
> - 레이더 합성정보: `…/typ01/cgi-bin/url/nph-rdr_cmp_inf?tm=YYYYMMDDHHMM&cmp=HSR&qcd=MSK&authKey=…`
> - 레이더 파일목록: `…/typ01/url/rdr_cmp_file_list.php?cmp=HSR&tm=YYYYMMDD&authKey=…`

## 2. "뽕 뽑기" — 케이스별 데이터 매핑

각 케이스가 **쓸 수 있는 레이어를 최대한** 끌어쓴다. 단, 교육적으로 의미 있는 것만(장식 금지).

| 케이스 | 종관일기도 | 위성 | 레이더 | 지상(ASOS/AWS) | SKEW-T/고층 | 앙상블 | 낙뢰 | 예보A/B | 실황 |
|---|---|---|---|---|---|---|---|---|---|
| ① 고기압 (easy) | 고기압 중심 명확, 전선 없음 | IR·VIS 구름 없음 | 무에코 확인 | 기압 높음·건조·맑음 | 안정 성층, CAPE≈0 | 좁은 분포(확신) | – | 일치 | 맑음 |
| ② 집중호우 (hard) | **전선 접근·기압골** | IR·VIS·**WV(수분유입 띠)** | HSR 시계열+**이동벡터**+강수량 | 기압 급강하·강수 급증 | **CAPE 높음·CIN 낮음·습윤** | **넓은 분포(불확실)** | 뇌우 동반 | **불일치** | 381.5mm |
| ③ 함정·에코 (trap) | 종관상 안정·전선 없음 | IR 엷은 구름 | **에코 존재** ← 핵심 | 강수 시작 신호 | 약한 불안정 | 맑음 예측(오답 유도) | – | 맑음(오답 유도) | 비 |

> **설계 의도**: 케이스마다 "결정적 레이어"가 다르다.
> ①은 종관+위성으로 쉽게 판단. ②는 SKEW-T 불안정도·수증기 채널·앙상블 스프레드가 핵심.
> ③은 종관·모델·앙상블이 모두 "맑음"이라고 해도 **레이더 에코 관측이 최우선**이라는 걸 배운다.
> 학습자가 케이스마다 **다른 레이어를 신뢰**해야 하고, 이게 6단계 체험 구조가 만드는 핵심 학습.

## 3. 화려한 인터랙티브 기능 카탈로그 (실데이터 기반)

각 기능은 **반드시 실데이터에 연결**된다. 화려함이 장식이 아니라 데이터 이해로 이어지게.

### I1. 레이더 타임머신 + 에코 이동벡터 ★대표 기능
- HSR 합성영상 6프레임을 스크럽/재생. 에코가 발달·이동하는 걸 직접 돌려봄.
- **이동속도 데이터로 화살표 벡터 오버레이** → "강수대가 어디로, 얼마나 빨리" 시각화.
- 현재 프레임 KST 타임스탬프 mono 실시간 갱신, 발달 구간 마커.

### I2. 위성 3채널 블렌드 슬라이더
- 적외↔가시↔**수증기** 채널을 슬라이더로 크로스페이드.
- 수증기 채널에서 "남서풍 수분 유입 띠"가 보이는 순간이 ②호우의 핵심 단서.
- 채널별 한 줄 AI 캡션("수증기영상의 밝은 띠 = 다량의 수분").

### I3. 단열선도(SKEW-T Log-P) 인터랙티브 ★핵심 기능 (독창성 1순위)
- 고층 레윈존데(라디오존데) 연직 데이터로 **대기 안정도 단면도**를 그림.
- 학습자가 고도를 짚으면 기온·이슬점 프로파일·불안정층 자동 강조.
- **CAPE / CIN / K-Index / Lifted Index** 자동 계산·표시 → "이 값이면 뇌우 가능" 해설.
- 바람 바브(wind barb) 오버레이로 연직 바람 시어 표시.
- 일반 기상 앱엔 없는 **진짜 예보관 도구 체험** = 심사 독창성 1순위.
- 출처: 기상청 API허브 고층관측 분야 (라디오존데 연직자료) ← **선택 아닌 필수 레이어**로 격상.

### I4. 멀티라인 동기화 차트 (ASOS)
- 기온·강수량·기압·습도를 한 타임축에 겹침.
- **기압 급강하 ↔ 강수 급증**이 동시에 일어나는 지점을 자동 하이라이트(상관 연출).
- 시간 슬라이더와 전 레이어 동기화(같은 시각을 모든 패널이 가리킴).

### I5. 레이어 합성 오버레이 뷰
- 위성 위에 레이더 반투명 겹침 + AWS 관측소 핀(값 라벨).
- 토글로 "관측의 층위"를 쌓아봄. 핀 호버 시 해당 지점 실측 팝오버.

### I6. 낙뢰 점멸 맵 (초단기예보 LGT)
- 낙뢰 위치를 시간 따라 점멸 애니메이션 → 뇌우 활성도 체감.

### I7. 강수확률 분포 입력 (앙상블 감각)
- 단일 숫자가 아니라 **확률 분포 슬라이더**로 "내 확신의 폭"을 표현 → Brier 직관 강화.

### I8. 전 레이어 타임 동기화 ("타임머신")
- 상단 마스터 시간 슬라이더 하나가 위성·레이더·ASOS·낙뢰를 **동시에 같은 시각**으로.
- "−6시간엔 이랬는데 지금은…"을 전방위로 재생 → 종관분석의 몰입 절정.

### I9. 결과 반전 연출 (build-spec §5-C C1 연동)
- 실황 공개 3비트 시퀀스 + 점수 카운트업 + 반사실 한 줄.

### I11. 종관 일기도 타임머신 ★실 예보관 첫 번째 도구
- 지상 일기도 → 850hPa → 500hPa 탭 전환 + 시간 슬라이더.
- 전선이 이동하고, 기압골이 접근하고, 제트류가 방향을 바꾸는 걸 시간축으로 재생.
- 실제 예보관이 분석을 시작할 때 제일 먼저 보는 자료 — 이걸 체험 첫 단계에 배치함으로써
  "예보는 큰 그림부터"라는 사고 순서를 자연스럽게 심어줌.
- 출처: 기상청 API허브 수치예보 분야 (GDAPS/LDAPS 일기도 이미지) ← 과거 아카이브 또는 캡처본.

### I12. 앙상블 스프레드 시각화
- 단기예보 앙상블 멤버들의 강수 예측 분포를 박스 플롯 또는 바이올린 플롯으로.
- "모델이 확신하면 좁은 분포, 불확실하면 넓은 분포" — 불확실성을 숫자가 아닌 그래프로 체감.
- 앙상블이 넓을 때 Brier 점수를 보수적으로 내야 이득이라는 전략을 시각으로 학습.
- 출처: 기상청 API허브 수치예보 앙상블 분야.

### I10. 데이터 구동 대기 모션 레이어 ★기상 컨셉의 정체성
> 기후·기상이 주제인 만큼 **대기 자체가 살아 움직이는 화면**이 있어야 한다.
> 단, 랜덤 장식이 아니라 **모든 입자·흐름이 실데이터 값에서 나온다.** 이것이 "화려함"과 "AI 웹 냄새"를 가르는 선.

| 모션 | 구동하는 실데이터 | 규칙 |
|---|---|---|
| **강수 입자(비/눈)** | ASOS/실황 강수량(mm) → 입자 밀도·속도, PTY → 비/눈 형태 | 381.5mm=폭우급 밀도, 0mm=없음. 색 `--rain` |
| **바람결 스트림라인** | ASOS 풍향·풍속 → 흐름 방향·속도 | WebGL/Canvas 라인, 강풍일수록 길고 빠르게 |
| **구름/안개 드리프트** | 위성 운량·수증기 채널 밝기 → 농도 | 맑음 케이스=옅게, 호우=짙게 |
| **번개 플래시** | 낙뢰 LGT 발생 → 화면 가장자리 순간 플래시 | 과용 금지, 실제 발생 시각에만 |
| **케이스 무드 필드** | 난이도/상황 → 배경 대기 톤 | easy=고요한 햇살 일렁임, hard=어둑·빗줄기 |

**클래스 유지 규칙(이게 없으면 그냥 AI 웹 됨):**
- **데이터 구동 필수** — 입자 수·방향·색이 실측값에 매핑. 난수 단독 생성 금지.
- **콘텐츠 뒤에만** — 판단해야 할 데이터 패널 위는 절대 침범, opacity ≤ 0.15, z-index 배경.
- **단색 팔레트** — 디자인 토큰(§UX 1.1)색만. 무지개·네온 금지.
- **성능 예산** — Canvas/WebGL, 입자 상한(모바일 ≤ 1.5k), 60fps 유지, 저사양 자동 감쇄.
- **하나의 일관된 계system** — 효과 짬뽕 금지. "대기 시뮬레이션" 한 세트.
- **reduced-motion 존중** — 끄면 정적 그라데이션으로 폴백.

> 결과 공개(I9)에서 호우 케이스는 **실제 381.5mm에 비례한 폭우 입자**가 쏟아지며 실황을 덮는다 →
> "수치"가 "체감"으로 바뀌는 순간. 이게 데이터 구동 모션의 교육적 정당성이자 가장 화려한 장면.

> 화려함의 원칙: **모든 인터랙션·모션은 실데이터에 묶인다.** 데이터와 무관한 장식 파티클·트랜지션만 금지(안티-제너릭 §11). 날씨 모션은 오히려 적극 권장.

## 5. Day 3 검증 우선순위 (막히는 곳부터)
실데이터 계획의 생사는 Day 3 검증에 달림. 순서:
1. ✅ 단기예보 `getVilageFcst` 과거 2발표시각 조회 가능 여부(모델 불일치 핵심)
2. 레이더 HSR 합성영상 + 파일목록 + 이동속도 (대표 기능 I1)
3. 위성 GK2A 3채널 과거 날짜 수신 (I2)
4. ASOS 시간/일자료 (차트·정답)
5. 고층 레윈존데 연직자료 (I3 단열선도)
6. 초단기 낙뢰 LGT (I6)

> 각 항목 **실호출 1회로 응답 형식 확정** 후 build-spec §4.1 표를 ✅로 갱신.
> 막히면 §16 리스크: 영상은 공공포털 백업 경로, 고층/낙뢰는 해당 케이스에서만 선택적 사용.
