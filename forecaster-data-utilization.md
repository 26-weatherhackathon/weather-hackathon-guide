# 데이터 활용 극대화 & 화려한 인터랙션 계획 (Data & Interaction Spec v1.0)

> 기상·기후 AI 해커톤 2026 출품작 · **"공공데이터 뽕 뽑기" + 인터랙티브 연출 설계**
> 목표: 기상청이 공개한 데이터를 **최대한 많이, 정면으로** 쓰고(주제적합성 20점),
> 그걸 **만지고 싶은 화려한 인터랙션**으로 만든다(체험형 25점·독창성 15점).
> 함께 보기: `forecaster-build-spec.md`, `forecaster-ux-spec.md`

## 1. 가용 데이터 전수 인벤토리 (무엇이 있는가)

기상청은 두 창구로 데이터를 공개한다. **둘 다 쓴다.**

### 1.1 기상청 API허브 (apihub.kma.go.kr · authKey) — 12개 분야
원시·고해상도 데이터와 **영상 파일**까지 주는 본진. 우리 케이스 영상의 출처.

| 분야 | 우리가 쓰는 데이터 | 케이스 활용 |
|---|---|---|
| **위성** | GK2A 16채널 — 적외(IR)·가시(VIS)·**수증기(WV)** + 산출물 46종 | 3채널 토글, 수증기로 호우 수분 유입 읽기 |
| **레이더** | HSR 합성영상 + 강수량(HSR) + **에코 이동속도** | 타임라인 스크럽 + 이동벡터 오버레이 |
| **지상관측** | ASOS(종관)·AWS(방재) 기온·강수·바람·습도·기압 | 멀티라인 추세 + 촘촘한 격자 핀 |
| **고층관측** | 레윈존데 연직 기온·습도·풍향풍속 | **단열선도(tephigram)** 대기안정도 |
| **해양관측** | 부이 파고·수온·파주기 | (태풍 확장 케이스) 해상 상태 |
| **태풍** | 태풍 정보·예상경로 | (확장 케이스) 경로 콘 시각화 |
| **수치모델** | 모델 격자 예측장 | 가이던스 보조(접근 가능 범위) |
| 예특보/응용/항공/세계/지진화산 | (직접 사용 낮음) | 확장 여지 |

> 실제 엔드포인트 예시(확인됨):
> - 위성: `apihub.kma.go.kr/api/typ05/api/GK2A/LE1B/{채널}/FD/data?date=YYYYMMDDHHMM&authKey=…`
> - 레이더 합성정보: `…/typ01/cgi-bin/url/nph-rdr_cmp_inf?tm=YYYYMMDDHHMM&cmp=HSR&qcd=MSK&authKey=…`
> - 레이더 파일목록: `…/typ01/url/rdr_cmp_file_list.php?cmp=HSR&tm=YYYYMMDD&authKey=…`

### 1.2 공공데이터포털 (data.go.kr · serviceKey) — JSON 예보·실황
정량 예보·실황을 깔끔한 JSON으로. 채점·판단 수치의 출처.

| 서비스 | 엔드포인트 | 주는 것 | 케이스 활용 |
|---|---|---|---|
| **단기예보** ✅검증됨 | `VilageFcstInfoService_2.0/getVilageFcst` | POP·PTY·TMX·PCP·SKY 3일 | 모델 A/B(2발표시각) |
| **초단기실황** | `…/getUltraSrtNcst` | 현재 대표 AWS 실황 | "지금" 시점 표시 |
| **초단기예보** | `…/getUltraSrtFcst` | 6시간 예보 + **낙뢰(LGT)** | 뇌우 위험·6h 흐름 |
| **ASOS 시간/일자료** | `AsosHourlyInfoService`·`AsosDalyInfoService` | 기온·강수·기압 등 | 추세 + 정답(실황) |
| **AWS(방재)** | `…AwsService` | 촘촘한 지점 관측 | 국지성 격자 핀 |
| **위성영상 조회** | data.go.kr 15058167 | 위성 영상(대안 경로) | API허브 백업 |
| **레이더영상/강수량(HSR)** | data.go.kr 15056924·15139446 | 레이더 영상·강수량 | API허브 백업 |
| **황사** | 황사 묶음 API | PM10·가시거리 | (황사 확장 케이스) |

> **이중화 전략**: 같은 데이터를 API허브(영상)·공공포털(JSON) 양쪽에서 확보 가능 → Day 3에 잘 되는 쪽으로 확정, 한쪽 막혀도 백업.

## 2. "뽕 뽑기" — 케이스별 데이터 매핑

각 케이스가 **쓸 수 있는 레이어를 최대한** 끌어쓴다. 단, 교육적으로 의미 있는 것만(장식 금지).

| 케이스 | 위성 | 레이더 | 지상 | 고층 | 낙뢰 | 예보A/B | 실황 |
|---|---|---|---|---|---|---|---|
| ① 고기압 맑음 (easy) | IR·VIS (구름 없음 확인) | 무에코 확인 | 기압 높음·건조 | 안정 성층 | – | 일치 | 맑음 |
| ② 집중호우 (hard) | IR·VIS·**WV(수분유입)** | HSR 시계열+**이동벡터**+강수량 | 기압급강하·강수급증 | **불안정·습윤** | 뇌우 동반 | **불일치** | 381.5mm |
| ③ 모델은 맑음·레이더 에코 (trap) | IR(엷은 구름) | **에코 존재** ← 핵심 | 강수 시작 신호 | 약한 불안정 | – | 맑음(오답 유도) | 비 |

> **설계 의도**: 케이스마다 "결정적 레이어"가 다르다. ②는 수증기·고층, ③은 레이더. 학습자가 **케이스마다 다른 데이터를 신뢰**하는 법을 배운다. 이것이 단순 시각화와의 결정적 차이.

## 3. 화려한 인터랙티브 기능 카탈로그 (실데이터 기반)

각 기능은 **반드시 실데이터에 연결**된다. 화려함이 장식이 아니라 데이터 이해로 이어지게.

### I1. 레이더 타임머신 + 에코 이동벡터 ★대표 기능
- HSR 합성영상 6프레임을 스크럽/재생. 에코가 발달·이동하는 걸 직접 돌려봄.
- **이동속도 데이터로 화살표 벡터 오버레이** → "강수대가 어디로, 얼마나 빨리" 시각화.
- 현재 프레임 KST 타임스탬프 mono 실시간 갱신, 발달 구간 마커.

### I2. 위성 3채널 블렌드 슬라이더
- 적외↔가시↔**수증기** 채널을 슬라이더로 크로스페이드.
- 수증기 채널에서 "남서풍 수분 유입 띠"가 보이는 순간이 ②호우의 핵심 단서.
- 채널별 한 줄 AI 캡션("수증기영상의 밝은 띠 = 다량의 수분").

### I3. 단열선도(Tephigram) 인터랙티브 ★독창성 포인트
- 고층 레윈존데 연직 데이터로 **대기 안정도 단면도**를 그림.
- 학습자가 고도를 짚으면 기온·이슬점·불안정층 강조. "왜 비구름이 발달하나"를 연직으로 이해.
- 일반 기상 앱엔 없는, **진짜 예보관 도구**를 체험 → 심사 임팩트.

### I4. 멀티라인 동기화 차트 (ASOS)
- 기온·강수량·기압·습도를 한 타임축에 겹침.
- **기압 급강하 ↔ 강수 급증**이 동시에 일어나는 지점을 자동 하이라이트(상관 연출).
- 시간 슬라이더와 전 레이어 동기화(같은 시각을 모든 패널이 가리킴).

### I5. 레이어 합성 오버레이 뷰
- 위성 위에 레이더 반투명 겹침 + AWS 관측소 핀(값 라벨).
- 토글로 "관측의 층위"를 쌓아봄. 핀 호버 시 해당 지점 실측 팝오버.

### I6. 낙뢰 점멸 맵 (초단기예보 LGT)
- 낙뢰 위치를 시간 따라 점멸 애니메이션 → 뇌우 활성도 체감.

### I7. 강수확률 분포 입력 (앙상블 감각)
- 단일 숫자가 아니라 **확률 분포 슬라이더**로 "내 확신의 폭"을 표현 → Brier 직관 강화.

### I8. 전 레이어 타임 동기화 ("타임머신")
- 상단 마스터 시간 슬라이더 하나가 위성·레이더·ASOS·낙뢰를 **동시에 같은 시각**으로.
- "−6시간엔 이랬는데 지금은…"을 전방위로 재생 → 종관분석의 몰입 절정.

### I9. 결과 반전 연출 (build-spec §5-C C1 연동)
- 실황 공개 3비트 시퀀스 + 점수 카운트업 + 반사실 한 줄.

> 화려함의 원칙: **모든 인터랙션은 "데이터를 더 잘 보게" 하는 목적이 있다.** 의미 없는 파티클·트랜지션 금지(안티-제너릭 §11).

## 4. 신청·인증·한도 (개발계획서 메모)

| 창구 | 절차 | 키 | 한도 |
|---|---|---|---|
| 기상청 API허브 | 회원가입→활용신청→authKey | authKey | 개인 일 2,000건 / 기업 3,000건 |
| 공공데이터포털 | 활용신청→serviceKey | serviceKey | 개발 일 10,000건, 운영 전환 시 증액 |

> **정적 사전수집이라 런타임 호출 0.** 수집은 개발 중 1회. 한도는 케이스 3개 수집에 전혀 문제 없음.

## 5. Day 3 검증 우선순위 (막히는 곳부터)
실데이터 계획의 생사는 Day 3 검증에 달림. 순서:
1. ✅ 단기예보 `getVilageFcst` 과거 2발표시각 조회 가능 여부(모델 불일치 핵심)
2. 레이더 HSR 합성영상 + 파일목록 + 이동속도 (대표 기능 I1)
3. 위성 GK2A 3채널 과거 날짜 수신 (I2)
4. ASOS 시간/일자료 (차트·정답)
5. 고층 레윈존데 연직자료 (I3 단열선도)
6. 초단기 낙뢰 LGT (I6)

> 각 항목 **실호출 1회로 응답 형식 확정** 후 build-spec §4.1 표를 ✅로 갱신.
> 막히면 §16 리스크: 영상은 공공포털 백업 경로, 고층/낙뢰는 해당 케이스에서만 선택적 사용.

## 6. 심사 배점 직결 요약
| 항목 | 배점 | 이 계획의 직결점 |
|---|---|---|
| 주제 적합성 | 20 | API허브 **6개 분야 + 공공포털 5개 서비스** 실사용, 원본 보존 증빙 |
| 체험·참여형 | 25 | I1~I9 만지는 인터랙션, 전 레이어 타임 동기화 |
| 아이디어 독창성 | 15 | **단열선도 등 진짜 예보관 도구** + 확률 분포 입력 + Brier |
| 실현 가능성 | 30 | 정적 사전수집(런타임 호출 0), 이중화 백업 |
| 학습 효과 | 10 | 케이스마다 다른 결정적 레이어 → 데이터 신뢰 판단력 |
