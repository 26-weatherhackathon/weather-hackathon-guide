## 0. 한 줄 요약

> **학습자를 기상청 예보관 의자에 앉혀, 진짜 기상청 데이터로 "내일 비 올까?"를 확률로 판단하게 하고,
> 실제 그날 무슨 일이 있었는지 까서 점수를 매기는 웹 체험 도구.**

- 데이터: **100% 실제 기상청 데이터** (위성·레이더·지상관측·예보·실황)
- 비주얼·해설: **AI 생성** (분위기 일러스트·예보관 캐릭터·챗봇) — 데이터와 명확히 구분 라벨
- 채점: **Brier Score** — 확신하고 틀리면 크게 감점, 애매하면 적당히 (진짜 예보 평가법)
- 분량: **3케이스 난이도 곡선** (쉬움 → 어려움 → 함정) + 성장 대시보드

## 1. 무엇을 만드나 — 사용자가 보는 화면 (체험 1회 흐름)

한 케이스는 **5단계**로 흐른다. 케이스당 6~9분.

```
[온보딩 1.5분]  날씨 데이터 4종이 뭔지 카드 4장 + 확인 퀴즈
      ↓
① 종관 분석   "2022-08-08 새벽, 너는 서울 예보관"
              위성사진·레이더를 시간 슬라이더로 돌려보며 큰 그림 파악
      ↓
② 모델 검토   기상청이 실제 낸 예보 2개를 나란히 비교
              (전날 23시 발표 vs 당일 08시 발표 — 강수확률이 다름!)
              → "어느 발표를 믿을래?" 선택
      ↓
③ 불확실성    지상관측값(강수량 급증·기압 하강) 추세를 보며 확신을 조정
      ↓
④ 최종 판단   슬라이더로 입력
              · 비 올 확률 (0~100%)
              · 강수 형태 (비/눈/비눈/없음)
              · 최고기온 (°C)
              · 근거 체크 (뭘 보고 정했나) — 1개 이상 필수
      ↓
⑤ 결과 공개   "그날, 실제로는…" → 381.5mm 폭우 공개
   + 채점     → Brier 점수 0→N 카운트업 연출
   + 리플렉션 → "무엇을 놓쳤나" 입력 → Claude가 맞춤 피드백
      ↓
[대시보드]    3케이스 점수를 추세선으로 연결 → 내 성장 + 약점 데이터 유형
```

**옆에 항상 떠 있는 것**: 우하단 AI 챗봇(WeatherAssistant). "이 빨간 에코 뭐예요?"
물어보면 지금 보는 데이터에 맞춰 설명 (정답은 안 알려줌).

### 3개 케이스 (난이도 곡선 = 학습 곡선)

| 순서 | 상황 | 난이도 | 학습 포인트 |
|---|---|---|---|
| 1 | 안정적 고기압, 맑음 | 쉬움 | 데이터 읽기 기초·성공 경험 |
| 2 | 2022-08-08 수도권 집중호우 (381.5mm) | 어려움 | 모델 불일치·확률적 사고 |
| 3 | 모델은 맑다는데 레이더엔 비구름 | 함정 | "데이터를 의심하라"·관측 우선 |

> 1로 자신감 → 2에서 갈등 → 3에서 반전. 대시보드가 세 점을 잇는다.

## 2. 왜 이렇게 만드나 — 핵심 설계 결정 3가지

| 결정 | 내용 | 이유 |
|---|---|---|
| **① 실데이터 정면 사용** | 위성·레이더·관측·예보를 진짜 기상청 API로 수집, 원본 JSON까지 보존 | 심사장에서 "이거 진짜 데이터예요"를 증빙. 주제 적합성 |
| **② AI 비주얼·챗봇 대량 투입** | 케이스당 이미지 15~20장 + 문맥 챗봇 | 딱딱한 데이터를 몰입형으로. 독창성·참여형 |
| **③ 3케이스 완전체** | 얕은 4개 대신, 난이도 곡선 3개를 완성도 100%로 | "원트릭 장난감"이 아닌 "커리큘럼"으로 보이게 |

### 두 가지 똑똑한 트릭

1. **"모델 불일치"를 공짜로 만든다**
   KIM/UM 같은 수치모델 원시 격자는 일반 참가자가 못 받는다. 대신 **같은 날짜를 두 발표시각으로
   호출**한다. 8/7 23시 발표와 8/8 08시 발표가 같은 시점을 다르게 예측한 차이 = "가이던스가 흔들린
   순간". 이건 100% 진짜 기상청 데이터다.

2. **모든 데이터를 미리 받아 정적 파일로 박제**
   런타임에 API를 안 부른다. 현장(천안) 네트워크가 끊겨도 무중단. CDN 캐시로 빠름. API 한도 걱정 0.

## 3. 무엇으로 만드나 — 기술 스택 (쉬운 설명)

| 역할 | 도구 | 한 줄 설명 |

## 5. 점수는 어떻게 매기나 — Brier Score

```
강수확률 (60%):  Brier = (내확률/100 − 실제0or1)²  →  점수 = 100 − round(Brier×100)
강수형태 (20%):  맞으면 100, 틀리면 0
최고기온 (20%):  1°C 빗나갈 때마다 −10점
```

| 내 예측 | 실제 | 점수 | 교훈 |
|---|---|---|---|
| "90% 비" | 비 옴 | ~99점 | 확신하고 맞으면 최고점 |
| "90% 비" | 맑음 | ~20점 | **확신하고 틀리면 폭망** |
| "50% 비" | 비 옴 | ~75점 | 애매하게 찍으면 안전빵 |

→ "모르면 50% 찍어"라는 비겁한 전략보다, **데이터로 확신을 높이는 게 이득**임을 점수로 체감시킨다.
대시보드에서 "매번 50%만 찍었다면" 가상선과 비교해 이걸 시각화.

## 6. AI는 어디에 쓰나 (데이터와 명확히 구분)

| 용도 | 무엇 | 원칙 |
|---|---|---|
| 분위기 비주얼 | 브리핑 일러스트·예보관 캐릭터·단계 배경·결과 연출 (케이스당 15~20장) | "분위기·연출"에만 |
| 리플렉션 피드백 | `api/reflect.js` — "무엇을 놓쳤나" 입력에 Claude가 맞춤 피드백 | 실데이터와 대조 |
| **문맥 챗봇** | `api/chat.js` + `WeatherAssistant.tsx` — 지금 보는 데이터를 설명 | **정답(예측 수치)은 안 알려줌** |

> 철칙: **판단 근거가 되는 수치는 100% 실측. AI는 분위기·해설·도우미.**
> 심사장에서 "데이터는 진짜, 비주얼은 AI"를 명확히 구분 설명 → 신뢰성 + 독창성 동시 확보.
