만들기

만들고자 하는 내용을 한국어로 설명하면 Claude가 코드를 작성하고 작업 저장소(GitHub)에 저장합니다. 아래 5단계는 초보자용 웹 인터페이스를 기준으로 하며, 터미널·IDE로 작업해도 됩니다.

Step 1 — 웹 Claude Code 시작하기
저장소는 개발용 작업 공간입니다

웹 Claude Code는 GitHub 저장소에 연결되어 동작합니다. 따라서 저장소를 먼저 만들지만, 이 저장소는 개발용 작업 공간이며 최종 소스는 GitHub이 아니라 구글 폼에 업로드합니다(Step 5).

1

GitHub 저장소 먼저 만들기

  1. github.com 로그인 후 우측 상단 + → New repository
  2. 이름 입력(예: weather-project) — 공개/비공개(Public/Private)는 자유롭게 선택
  3. Create repository 클릭
저장소 이름은 이후 Vercel 주소가 되므로, 팀 프로젝트 이름으로 지어 두면 좋습니다.
2

claude.ai/code 접속 후 GitHub 연결

  1. claude.ai/code 접속 → 제공받은 계정으로 로그인
  2. 입력창 아래쪽의 "+ 레포 선택..." 버튼 클릭
  3. 목록에서 방금 만든 저장소(예: weather-project) 선택
  4. 선택이 끝나면 버튼 자리에 저장소 이름과 브랜치 이름이 표시됩니다. 브랜치 이름은 계정 설정에 따라 main 또는 master로 보일 수 있으며, 둘 다 '기본 브랜치'라는 같은 의미이므로 그대로 두면 됩니다.
이 연결을 한 번 해 두면 이후 Claude Code가 코드를 작성할 때마다 GitHub에 자동으로 저장합니다.
참고 저장소 목록이 보이지 않거나 이후 저장(푸시)이 거부되면, Claude GitHub App(github.com/apps/claude)을 설치하고 해당 저장소에 접근·쓰기 권한을 부여해야 합니다. 확인 방법은 팀 협업 페이지의 'Push할 때 SSH 키 에러' 항목을 참고하세요.
Step 2 — Claude Code로 만들기
3

한국어로 요청하면 코드가 작성됩니다

요청할 때 (1) 무엇을 보여줄지 (2) 어떻게 조작할지 (3) 파일 이름(index.html), 이 세 가지만 포함하면 됩니다.

예시 대화
기온과 풍속 슬라이더로 체감온도를 보여주는 웹페이지를 index.html 하나로 만들어 줘. 기상청 체감온도 공식을 사용해 줘.
슬라이더를 움직이면 체감온도가 실시간으로 바뀌는 index.html을 만들었습니다. GitHub에 저장했습니다.
배경색을 하늘색으로 바꾸고, 모바일에서도 잘 보이게 해 줘.
배경색과 반응형 레이아웃을 적용했습니다. 변경 사항을 GitHub에 저장했습니다.
P

예시 프롬프트

대괄호 부분만 우리 팀 주제로 바꾸세요.

  • 기본형 — "[기온]과 [풍속] 슬라이더로 [체감온도]를 보여주는 웹페이지를 index.html 하나로 만들어 줘."
  • 응용형 — "[강수량] 슬라이더를 조절하면 [홍수 위험도]가 바뀌는 인터랙티브 웹페이지를 초등학생도 이해할 수 있게 만들어 줘. index.html 하나로."
  • 디자인 — "방금 페이지를 더 보기 좋게 다듬어 줘. 파란 하늘색 계열, 모바일에서도 잘 보이게."
  • 오류 해결 — "방금 페이지에서 [오류 메시지]가 나와. 원인을 알려주고 고쳐 줘."
팁 한 번에 완성하지 않아도 괜찮습니다. "여기를 이렇게 바꿔 달라"는 식으로 대화를 이어가며 다듬으세요. 수정할 때마다 GitHub에 자동 저장됩니다.
Step 3 — CLAUDE.md 설정하기
4

프로젝트 규칙서, CLAUDE.md

CLAUDE.md는 Claude Code가 프로젝트를 열 때 가장 먼저 읽는 파일입니다. 여기에 적어 둔 규칙은 매 대화마다 자동으로 적용됩니다. "파일 이름은 index.html로 고정", "한국어로 응답" 같은 규칙을 넣어 두면 매번 설명할 필요가 없습니다.

  1. 아래 CLAUDE.md 내용을 전체 복사합니다.
  2. Claude Code에 붙여넣으며 "이 내용으로 저장소 최상단에 CLAUDE.md 파일을 만들어 줘"라고 요청합니다.
  3. Claude가 파일을 생성해 GitHub에 저장합니다.
파일 이름은 반드시 CLAUDE.md(대문자)이며, 위치는 저장소 최상단 폴더입니다.
CLAUDE.md (복사용)# CLAUDE.md ## 기본 규칙 - 파일 이름은 index.html로 고정합니다. - 응답은 한국어로 합니다. - 코드를 바꾼 뒤에는 무엇을 왜 바꿨는지 한 줄로 알려줍니다. ## 안전 규칙 (최소) - 큰 변경을 하기 전에는 현재 동작하는 상태를 먼저 저장(커밋)해, 문제가 생기면 되돌릴 수 있도록 합니다. - 변경 후에는 페이지가 정상 동작하는지 확인한 뒤 저장합니다. ## 2인 이상 팀으로 참가할 때 - GitHub로 협업할 경우 브랜치를 나눠 개발하고 PR로 합칩니다. - 구체적인 협업 방법은 운영팀이 제공하는 별도 콘텐츠(영상·문서)를 따릅니다.
혼자(1인) 참가하는 경우 위 협업 규칙 없이 그대로 개발하면 됩니다. 팀원이 2명 이상이라면 GitHub를 활용한 협업 방법을 운영팀이 별도 영상·문서로 제공할 예정이니 트랙 03과 함께 참고하세요.
Step 4 — Vercel로 배포하기
5

GitHub로 로그인 후 프로젝트 가져오기

  1. vercel.com 접속 → Continue with GitHub
  2. Add New → Project 클릭
  3. 방금 만든 저장소 선택 → Import
6

Framework Preset 설정

Import 직후 설정 화면에서 Framework Preset을 선택합니다. 이 가이드처럼 index.html을 직접 만든 경우에는 Other를 선택하면 됩니다. 별도 빌드 과정이 없기 때문입니다.

주의 Preset이 맞지 않으면 빌드가 실패할 수 있습니다. index.html을 직접 만들었다면 반드시 Other입니다.
7

배포 후 링크 받기

  1. 설정 확인 후 Deploy 클릭
  2. 1~2분 뒤 weather-project.vercel.app 같은 주소가 생성됩니다
  3. 이 주소가 구동 URL(제출 항목 ①)입니다
참고 GitHub에서 index.html을 수정하면 Vercel이 자동으로 다시 배포하므로, 별도로 재배포할 필요가 없습니다.
배포 후 확인

다른 컴퓨터에서도 링크가 열리는지, 슬라이더 등 인터랙티브 기능이 정상 작동하는지 확인하세요. 구동 환경은 크롬 브라우저 기준이며, 1366×768·1920×1080·2560×1440 해상도에서 정상 표시되어야 합니다.

Step 5 — 제출하기
8

제출 항목 4가지

제출은 아래 4가지입니다. 웹으로 만들었든 터미널로 만들었든 동일하게 적용됩니다.

  • ① 구동 가능한 URL — 누구나 접속해 바로 조작할 수 있는 웹 주소입니다. Vercel 배포를 권장하며 GitHub Pages·Netlify 등 다른 배포도 인정됩니다. 로그인·결제 없이 열려야 하며, 구동 환경은 크롬 브라우저(해상도 1366×768·1920×1080·2560×1440 지원) 기준입니다.
  • ② 소스 코드 (구글 폼 업로드) — 완성된 소스를 압축(zip)해 운영팀이 안내하는 구글 폼에 업로드합니다. GitHub 저장소 화면의 Code → Download ZIP으로 zip을 받으면 편리합니다. GitHub 링크 제출이 아니라 파일 업로드이며, node_modules·빌드 산출물은 제외하고 소스만 담으세요. (구글 폼 주소: 운영팀 별도 안내)
  • ③ 주요 프롬프트 세션 — 제공받은 팀 계정에서 개발에 사용한 대화 세션의 공유 링크를 제출합니다. AI 활용의 진정성을 확인하기 위한 자료입니다. 개발을 팀 대표 계정에 모아 진행하면 세션 정리·제출이 간편합니다. (세션 공유 방법: 운영팀 별도 안내)
  • ④ 구동 환경 / 배포 방법 문서 (README 등) — 다른 사람이 그대로 실행·배포할 수 있도록 정리합니다. 정적 페이지(index.html)는 "브라우저에서 열면 됨" 수준이면 충분하고, 프레임워크(React/Next/Astro 등)나 빌드 도구(Vite 등)를 사용했다면 설치·실행·빌드·배포 명령을 단계로 적습니다. 사용한 외부 리소스(라이브러리·이미지·폰트·데이터)의 출처·라이선스도 함께 명시합니다.
✓

제출 전 체크리스트

  • URL이 크롬 브라우저에서 열리고 인터랙션이 정상 동작하는가
  • 소스 zip을 구글 폼에 올렸는가 (node_modules·빌드 산출물 제외)
  • 개발에 사용한 세션의 공유 링크를 준비했는가
  • README에 실행/배포 방법과 외부 리소스 출처·라이선스가 적혀 있는가
